# ToDoList 使用说明

ToDoList 是一款智能待办工具，帮助您管理任务、按 **四象限** 区分优先级，并用 AI 推荐今日最值得做的 **Top 3**。

---

## 一、快速开始

1. 解压或复制整个程序文件夹到任意位置。
2. 双击 **ToDoList.exe** 或 **启动 ToDoList.bat** 运行。
3. 首次运行会自动在程序目录下创建 **data** 文件夹，用于保存任务和日志。
4. 建议屏幕分辨率 **1280×720** 及以上；Windows **10 / 11 64 位**。

---

## 二、界面说明

### 主窗口

| 区域 | 说明 |
|------|------|
| **标题栏** | 最小化、最大化/还原（F11）、关闭 |
| **工具栏** | 常用功能按钮（见下文） |
| **左侧任务列表** | 所有进行中的任务 |
| **右侧四象限** | Q1～Q4 看板，拖拽可调整象限 |
| **AI Top 3** | 今日优先推荐的 3 条任务及理由 |

### 工具栏按钮

| 按钮 | 做什么 |
|------|--------|
| **今日任务** | 一次输入多条今日待办（每行一条） |
| **添加任务** | 添加单条任务 |
| **删除任务** | 勾选多条后批量删除 |
| **AI 分析优先级** | 让 AI 划分象限并更新 Top 3 |
| **重置象限** | 清空所有象限，需重新 AI 分析 |
| **设置 LLM** | 配置 AI 模型（见第四节） |
| **历史记录** | 查看往日任务快照 |
| **每日评估** | 查看 AI 对每日完成情况的总结 |
| **设置** | 密码、快捷键等（见第五节） |

## 三、功能说明

### 3.1 添加与编辑任务

**添加单条任务**

1. 点击 **添加任务**。
2. 填写 **标题**（必填）。
3. 选择 **截止日期**。
4. 象限可选「不确定 (由 AI 划分)」或手动指定 Q1～Q4。
5. 点击 **确定**。

**批量添加今日任务**

1. 点击 **今日任务**（或按全局快捷键，默认 **Alt+Shift+J**）。
2. 在文本框中 **每行写一条** 任务。
3. 可点击 **导入昨日未完成任务** 把昨天没做完的带过来。
4. 点击 **添加**。已有任务会保留，只追加新内容。

**编辑任务：** 双击任务标题即可修改。

**标记完成：** 在列表或四象限看板中勾选 **完成** 复选框。

**删除任务：**

- 批量：点 **删除任务** → 勾选 → **确认删除**。
- 单条：在任务上 **右键 → 删除此任务**。

---

### 3.2 四象限

| 象限 | 含义 |
|------|------|
| **Q1** | 重要且紧急 |
| **Q2** | 重要不紧急 |
| **Q3** | 紧急不重要 |
| **Q4** | 不重要不紧急 |

- 表格里显示 **待AI** 表示还没划分象限，需要做一次 **AI 分析优先级**。
- 在四象限面板里 **拖拽** 任务可更换象限。
- **重置象限** 会把所有任务改回「不确定」，之后需重新分析。

---

### 3.3 AI 分析优先级

1. 先添加一些任务。
2. 点击 **AI 分析优先级**。
3. 等待分析完成（界面会提示「分析中请稍后」）。
4. 完成后：未分象限的任务会自动归入 Q1～Q4，右侧 **Top 3** 和理由会更新。

**说明：**

- 若 AI 暂时不可用，程序会用 **内置规则** 给出 Top 3，不影响基本使用。
- 可在 **设置 → 工具 → AI 分析过程** 查看最近一次分析的详细记录（需先成功分析过一次）。

---

### 3.4 Top 3 推荐

- **主窗口右侧**：随时查看 Top 3 列表；点击某条可看推荐理由。
- **Top 3 弹窗**：按 **Alt+Shift+3**（可在设置里改），在屏幕旁弹出小窗快速查看。
- 关闭或最小化程序前，Top 3 会自动保存。

---

### 3.5 历史记录

点击 **历史记录**，按日期查看：

- 当天的任务快照；
- 已过期自动归档的任务。

只读浏览，不能修改历史数据。

---

### 3.6 每日评估

点击 **每日评估**，查看程序对某一天任务完成情况的总结（完成数、到期数、评价文字等）。

- 每天 **23:59** 后会自动生成；
- 若某天漏生成，下次启动时会补做。

---

### 3.7 系统托盘

关闭窗口时可选择 **最小化到系统托盘**，程序在后台继续运行。

**托盘图标右键菜单：**

| 菜单 | 功能 |
|------|------|
| 显示主窗口 | 重新打开主界面 |
| 今日任务 | 打开今日任务对话框 |
| 添加任务 | 打开添加任务对话框 |
| Top 3 弹窗 | 弹出 Top 3 小窗 |
| 退出 | 完全退出程序 |

单击或双击托盘图标也可 **显示主窗口**。

---

### 3.8 全局快捷键

默认快捷键（可在 **设置 → 快捷键** 中修改）：

| 快捷键 | 功能 |
|--------|------|
| **Alt+Shift+J** | 显示主窗口并打开「今日任务」 |
| **Alt+Shift+3** | 弹出 Top 3 窗口 |

录制方法：点击输入框，按下想要的组合键。须包含 **Ctrl / Alt / Shift / Win** 之一。

---

### 3.9 密码与锁屏（可选）

在 **设置 → 安全** 中可设置：

| 设置 | 说明 |
|------|------|
| 登录密码 | 至少 4 位 |
| 启动时需密码 | 每次打开程序要先解锁 |
| 系统锁屏后锁定 | Windows 锁屏后多久锁定本程序（0 = 立即） |
| 空闲自动锁定 | 无操作多少分钟后锁定（0 = 关闭） |
| 关闭窗口时最小化到托盘 | 勾选后，Alt+F4 等会直接最小化到托盘 |

锁屏界面可 **解锁** 或 **退出** 程序。

---

### 3.10 关闭程序

点击标题栏 **×** 时可选：

| 选项 | 说明 |
|------|------|
| **最小化到系统托盘** | 后台继续运行（默认） |
| **退出** | 完全关闭 |
| **取消** | 不关闭 |

---

### 3.11 数据在哪里

任务和日志都在程序目录下的 **data** 文件夹：

- `tasks.db` — 任务数据库  
- `tasks-日期.json` — 每日快照  
- `logs/` — 运行日志  

**换电脑：** 复制整个程序文件夹（含 data）即可带走全部任务。

## 四、AI 模型配置

使用 **AI 分析优先级** 前，需先配置大语言模型。点击工具栏 **设置 LLM**。

### 4.1 对话框说明

| 项 | 说明 |
|----|------|
| **提供商** | 选择 AI 服务类型 |
| **Base URL** | 服务地址 |
| **API Key** | 云端服务的密钥（本地 Ollama 一般留空） |
| **模型** | 模型名称 |
| **测试连接** | 检查能否连通 |
| **保存** | 保存并启用 |

配置成功后，工具栏右侧会显示当前 Provider 和 Model。

---

### 4.2 方式一：本地 Ollama（推荐离线使用）

适合在本机运行 AI、不上传数据到云端。

**第一步：安装 Ollama**

1. 打开 [https://ollama.com](https://ollama.com) 下载并安装。
2. 安装完成后，Ollama 会在后台运行（默认端口 **11434**）。

**第二步：下载模型**

打开命令提示符或 PowerShell，执行（示例）：

```
ollama pull qwen2.5:3b
```

也可选用其他已支持的模型，记住模型名称。

**第三步：在 ToDoList 中配置**

1. 点击 **设置 LLM**。
2. **提供商** 选 **Ollama (本地 Qwen)**。
3. **Base URL** 填：`http://127.0.0.1:11434`（一般不用改）。
4. **API Key** 留空。
5. **模型** 填：`qwen2.5:3b`（与上一步 pull 的名称一致）。
6. 点 **测试连接**，成功后再点 **保存**。

**常见问题：**

- 测试失败 → 确认 Ollama 已启动，浏览器能打开 `http://127.0.0.1:11434`。
- 模型不存在 → 重新执行 `ollama pull 模型名`。

---

### 4.3 方式二：DeepSeek 云端

适合有网络、希望用云端大模型的用户。

**第一步：获取 API Key**

1. 注册 [DeepSeek 开放平台](https://platform.deepseek.com)。
2. 创建 API Key 并复制保存。

**第二步：在 ToDoList 中配置**

1. 点击 **设置 LLM**。
2. **提供商** 选 **DeepSeek**。
3. **Base URL**：`https://api.deepseek.com/v1`（默认即可）。
4. **API Key**：粘贴您的密钥。
5. **模型**：`deepseek-chat`（默认即可）。
6. **测试连接** → **保存**。

---

### 4.4 方式三：Kimi（Moonshot）

**第一步：** 在 [Moonshot 开放平台](https://platform.moonshot.cn) 注册并获取 API Key。

**第二步：在 ToDoList 中配置**

1. **提供商** 选 **Kimi (Moonshot)**。
2. **Base URL**：`https://api.moonshot.cn/v1`。
3. **API Key**：填写密钥。
4. **模型**：`moonshot-v1-8k`（或平台提供的其他模型名）。
5. **测试连接** → **保存**。

---

### 4.5 方式四：其他 OpenAI 兼容接口

若使用 OpenAI 或兼容 OpenAI 格式的第三方服务：

1. **提供商** 选 **Custom OpenAI**。
2. **Base URL**：服务商给的 API 地址（通常以 `/v1` 结尾）。
3. **API Key**：按服务商要求填写。
4. **模型**：按服务商文档填写模型名。
5. **测试连接** → **保存**。

---

### 4.6 配置完成后

1. 添加若干任务。
2. 点击 **AI 分析优先级** 验证是否正常工作。
3. 若失败：回到 **设置 LLM** 检查配置；暂时仍可使用规则层 Top 3 和全部任务管理功能。

---

## 五、常见问题

**Q：AI 分析失败怎么办？**  
先 **设置 LLM → 测试连接**。Ollama 用户确认服务已启动、模型已下载；云端用户检查 API Key 和网络。

**Q：快捷键无效？**  
到 **设置 → 快捷键** 重新设置并保存；避免与别的软件占用同一组合键。

**Q：任务会丢吗？**  
数据在 **data** 文件夹。定期备份整个程序目录即可。

**Q：忘记密码？**  
程序不提供在线找回。任务数据在 data 里不会丢，但需自行处理密码相关设置（如联系管理员或清除程序配置后重设）。
