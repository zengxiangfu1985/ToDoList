## 1. 产品简介

ToDoList 是一款基于 **艾森豪威尔矩阵（四象限）** 的智能待办管理工具，支持：

- 任务的添加、编辑、批量导入、删除与完成跟踪
- 使用 **大语言模型（LLM）** 自动划分象限并推荐 **Top 3** 优先任务
- LLM 不可用时自动降级为 **规则层评分**，保证基本可用
- **行为学习引擎** 根据您的操作逐步优化评分权重
- **系统托盘** 驻留、**全局快捷键**、可选 **密码锁屏**
- 可选 **Microsoft 365 / Outlook** 旗标邮件同步为任务
- 每日任务快照、过期归档、AI 每日完成评估

## 2. 安装与启动

### 2.1 绿色免安装版

1. 解压 `ToDoList-Portable-*.zip` 到任意目录（如桌面、U 盘）。
2. 双击 `ToDoList.exe` 或 `启动 ToDoList.bat` 运行。
3. **无需** 单独安装 Qt、.NET 或其他运行库。

### 2.2 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11，64 位 |
| 屏幕 | 建议 1280×720 及以上（程序会自动适配分辨率） |
| 磁盘 | 程序目录需可写（用于 `data` 数据库与日志） |
| 本地 AI（可选） | [Ollama](https://ollama.com)，默认 `http://127.0.0.1:11434` |
| 云端 AI（可选） | 可访问 HTTPS 的 DeepSeek / Kimi / OpenAI 兼容 API |
| M365（可选） | Azure 应用注册 + 微软账号 |

### 2.3 首次运行

- 程序会在可执行文件同级自动创建 `data` 目录存放任务与日志。
- 若已在「设置」中启用「启动时需密码」，首次显示主窗口后会进入锁屏登录界面。

## 3. 主界面概览

主窗口标题：**ToDoList — AI 智能待办**。采用自定义标题栏（无边框窗口）。

### 3.1 标题栏

| 按钮 | 功能 |
|------|------|
| **—** | 最小化窗口 |
| **□ / ❐** | 最大化 / 还原（也可按 **F11**） |
| **×** | 关闭（弹出关闭方式选择，见第 13 节） |

### 3.2 工具栏

工具栏按钮从左到右：

| 按钮 | 说明 |
|------|------|
| **今日任务** | 批量输入今日待办，可导入昨日未完成项 |
| **添加任务** | 添加单条任务，可手动指定象限 |
| **删除任务** | 进入批量删除模式 |
| **取消** | 退出批量删除模式（仅删除模式下显示） |
| **AI 分析优先级** | 调用 LLM 划分象限并生成 Top 3 |
| **重置象限** | 将所有任务象限重置为「不确定」 |
| **设置 LLM** | 配置 AI 提供商与模型 |
| **历史记录** | 查看每日任务快照与过期归档 |
| **每日评估** | 查看 AI 每日完成情况评估 |
| **设置** | 密码、锁屏、快捷键、工具、关于 |

工具栏右侧显示当前 LLM 状态，例如：`Provider: Ollama/Qwen | Model: qwen2.5:3b`。

窄屏下工具栏支持横向滚动，避免按钮被裁切。

### 3.3 主内容区

- **左侧：任务列表**  
  列包括：序号、标题、截止、象限、分数、完成。支持双击编辑、右键菜单、勾选完成。

- **右侧：四象限 + AI 推荐**  
  - **Eisenhower 四象限**：Q1～Q4 看板，可拖拽任务、勾选完成。  
  - **AI Top 3 推荐**：列表展示优先任务，下方显示推荐理由。

### 3.4 视图菜单（帮助栏下方）

| 菜单项 | 快捷键 |
|--------|--------|
| 最小化 | **Ctrl+M** |
| 最大化 / 还原 | **F11** |

## 4. 任务管理

### 4.1 添加任务

**入口：** 工具栏 **添加任务**，或托盘菜单 **添加任务**。

| 字段 | 说明 |
|------|------|
| **标题** | 必填 |
| **截止日期** | 日历选择，保存为当日 23:59 |
| **象限** | 可选「不确定 (由 AI 划分)」或手动指定 Q1～Q4 |

### 4.2 今日任务（批量）

**入口：** 工具栏 **今日任务**，或全局快捷键（默认 **Alt+Shift+J**）。

- 在文本框中 **每行输入一条** 任务标题。
- 新任务截止时间为 **今天 23:59**，象限默认为「由 AI 划分」。
- **已有今日任务会保留**，本次仅追加新行，不会覆盖。
- 点击 **导入昨日未完成任务** 可将昨日未完结项加入列表。
- 首次打开时，若检测到昨日有未完成任务，会询问是否导入。

### 4.3 编辑任务

- **双击** 任务行的标题列，打开编辑对话框（字段同「添加任务」）。

### 4.4 完成任务

- 在任务列表 **完成** 列勾选，或在四象限看板中勾选任务前的复选框。
- 已完成任务显示删除线样式；再次取消勾选可恢复为未完成。

### 4.5 删除任务

**批量删除：**

1. 点击 **删除任务**，表格出现 **选择** 列。
2. 勾选要删除的任务，按钮变为 **确认删除**。
3. 确认后不可撤销。点击 **取消** 退出删除模式。

**单条删除：** 在任务行 **右键 → 删除此任务**。

### 4.6 过期任务

- 截止已过的任务会自动归档到 `expired-YYYY-MM-DD.json`，并从活动列表中移除。
- 可在 **历史记录** 中查看。

## 5. 艾森豪威尔四象限

### 5.1 四象限含义

| 象限 | 名称 | 说明 |
|------|------|------|
| **Q1** | 重要且紧急 | 立即处理 |
| **Q2** | 重要不紧急 | 计划安排 |
| **Q3** | 紧急不重要 | 尽量委派或快速处理 |
| **Q4** | 不重要不紧急 | 可延后或删除 |

### 5.2 看板操作

- 仅 **已分配象限** 的任务显示在四象限面板中。
- 表格中象限为 **待AI** 表示尚未划分，需执行「AI 分析优先级」。
- **拖拽** 任务可在象限之间移动，状态栏会提示「任务已移至 Qn」。
- 象限内勾选框与左侧任务列表的完成状态 **同步**。

### 5.3 重置象限

点击 **重置象限** 后，所有任务变为「不确定 (由 AI 划分)」，需重新执行 AI 分析才会再次划分。

## 6. AI 分析优先级与 Top 3

### 6.1 AI 分析优先级

**入口：** 工具栏 **AI 分析优先级**。

**流程：**

1. 若无任务，提示「请先添加任务」。
2. 分析开始后，界面显示 **「分析中请稍后」** 遮罩，按钮暂时禁用。
3. 分析完成后：
   - 为「不确定」象限的任务自动分配 Q1～Q4；
   - 更新右侧 **AI Top 3 推荐** 及推荐理由；
   - 状态栏显示划分数量等信息。

**LLM 不可用时的降级：**

- 仍可使用 **规则层** 生成 Top 3 排序；
- 状态栏 / 系统托盘可能显示警告提示。

### 6.2 Top 3 推荐（主面板）

- 列表格式：`#排名 标题 (分数)`。
- 点击列表项，下方 **推荐理由** 区域显示 AI 说明。
- 当日 Top 3 会按 LLM 提供商与模型自动保存，切换模型后可能需重新分析。

### 6.3 Top 3 弹窗

**入口：**

- 全局快捷键（默认 **Alt+Shift+3**）；
- 托盘菜单 **Top 3 弹窗 (快捷键)**。

- 在鼠标附近弹出置顶小窗，展示 Top 3 及理由。
- 若无 AI 结果，使用规则层 Top 3。
- 若无任何待办，托盘提示「暂无待办任务」。

### 6.4 AI 分析过程

**入口：** **设置 → 工具 → AI 分析过程**（需先执行过一次 AI 分析）。

- 查看最近一次分析的摘要与详细步骤（含 LLM 请求/响应记录）。
- 也可通过 **帮助 → 打开 AI 分析记录文件** 用外部编辑器查看 `ai-analysis-latest.log`。

### 6.5 行为学习

- 程序会根据您变更象限、完成任务、采纳 AI 推荐等操作，自动调整评分权重。
- 状态栏短暂显示类似：`学习权重 U:… I:… B:… E:…`（紧急/重要/业务/精力四维）。

## 7. LLM 配置

**入口：** 工具栏 **设置 LLM**。

### 7.1 对话框字段

| 字段 | 说明 |
|------|------|
| **提供商** | Ollama (本地 Qwen) / DeepSeek / Kimi (Moonshot) / Custom OpenAI |
| **已保存模型** | 切换当前提供商下曾测试成功的配置 |
| **Base URL** | API 基址 |
| **API Key** | 云端必填；Ollama 本地通常留空 |
| **模型** | 模型名称 |
| **测试连接** | 验证配置；成功后加入「已保存模型」列表 |
| **保存** / **取消** | 保存后立即切换运行时 Provider |

### 7.2 各提供商默认参数

| 提供商 | 默认 Base URL | 默认模型 |
|--------|--------------|---------|
| Ollama | `http://127.0.0.1:11434` | `qwen2.5:3b` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| Kimi | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` |
| Custom OpenAI | `https://api.openai.com/v1` | `gpt-4o-mini` |

### 7.3 Ollama 本地配置步骤

1. 安装 [Ollama](https://ollama.com) 并确保服务运行。
2. 在终端执行：`ollama pull qwen2.5:3b`（或您使用的其他模型）。
3. 在 LLM 设置中选择 **Ollama**，Base URL 保持 `http://127.0.0.1:11434`，填写对应模型名。
4. 点击 **测试连接**，成功后 **保存**。

### 7.4 云端 API 配置步骤

1. 在对应平台注册并获取 **API Key**。
2. 选择 DeepSeek / Kimi / Custom OpenAI 提供商。
3. 填写 Base URL、API Key、模型名。
4. **测试连接** → **保存**。

## 8. 设置

**入口：** 工具栏 **设置**。

设置窗口内容较多时支持 **滚动查看**；底部按钮始终固定可见。

### 8.1 安全

| 设置项 | 说明 |
|--------|------|
| **当前密码** | 修改或清除密码时填写 |
| **新密码** | 至少 4 位；留空表示不修改 |
| **确认新密码** | 须与新密码一致 |
| **系统锁屏后锁定** | 0～120 分钟；**0 = Windows 锁屏后立即锁定**；默认 5 分钟 |
| **空闲自动锁定** | 0～480 分钟；**0 = 不启用**；监测键鼠与滚轮活动 |
| **启动时需密码** | 程序启动时要求输入密码（须先设置密码） |
| **关闭行为** | 勾选「关闭窗口时最小化到托盘」（默认开启） |

**操作按钮：**

- **立即锁定** — 需已设密码，立刻进入锁屏。
- **清除密码** — 需验证当前密码；清除后关闭所有锁定功能。
- **保存** / **取消**

### 8.2 快捷键

| 功能 | 默认快捷键 |
|------|-----------|
| **今日任务** | `Alt+Shift+J` |
| **Top 3 弹窗** | `Alt+Shift+3` |

- 点击输入框后按下组合键即可录制。
- 须包含 **Win / Ctrl / Alt / Shift** 之一，且绑定字母、数字或功能键。
- 两个快捷键 **不能相同**。
- 保存后自动重新注册；若被其他程序占用，状态栏会提示。

### 8.3 工具

| 按钮 | 功能 |
|------|------|
| **AI 分析过程** | 查看最近一次 AI 分析详情 |
| **Microsoft 365** | Outlook 旗标邮件同步（见第 10 节） |

### 8.4 关于

- **版本号**：显示程序编译日期。

## 10. Microsoft 365 集成

**入口：** **设置 → 工具 → Microsoft 365**。

### 10.1 前置准备（Azure 应用注册）

1. 登录 [Azure 门户](https://portal.azure.com) → **Microsoft Entra ID** → **应用注册** → **新注册**。
2. 记录 **应用程序(客户端) ID**。
3. 在 **身份验证** 中启用 **允许公共客户端流**（设备代码流需要）。
4. 在 **API 权限** 中添加：`User.Read`、`Mail.Read`、`offline_access`，并同意授权。

### 10.2 程序内配置

| 字段 | 说明 |
|------|------|
| **Tenant ID** | 填 `common`（个人/多租户）或组织租户 GUID |
| **Client ID** | Azure 应用的客户端 ID |
| **设备码登录** | 发起 OAuth 设备码授权 |
| **验证 URL / 用户码** | 在浏览器中完成登录时使用 |
| **复制 / 打开浏览器** | 辅助完成授权 |
| **确认已授权** | 浏览器登录完成后点击，换取 Token |
| **同步 Outlook 旗标邮件为任务** | 导入最多 25 封旗标邮件 |

### 10.3 同步规则

- 邮件 **主题** → 任务标题。
- **截止** = 收件时间 + 1 天。
- 默认象限 **Q2**，来源标记 `outlook`。
- 相同外部 ID 的邮件不会重复导入。

## 11. 锁屏与密码

### 11.1 设置密码

1. 打开 **设置 → 安全**。
2. 填写 **新密码** 与 **确认新密码**（至少 4 位）。
3. 点击 **保存**。新设密码后默认开启「启动时需密码」。

### 11.2 锁屏界面

全屏深色界面，包含：

- **密码输入框**
- **解锁**（也可按 **Enter**）
- **退出**（保存数据后退出程序）

### 11.3 触发锁屏的方式

| 方式 | 说明 |
|------|------|
| 启动登录 | 「启动时需密码」已启用 |
| 立即锁定 | 设置对话框中点击 **立即锁定** |
| 系统锁屏 | Windows 锁屏后，按设定延迟锁定 |
| 空闲锁定 | 超过设定分钟无操作 |

锁屏期间主界面与状态栏隐藏，保护隐私。

## 12. 历史记录与每日评估

### 12.1 历史记录

**入口：** 工具栏 **历史记录**。

- 左侧：日期列表（来自每日快照 `tasks-*.json` 与过期归档 `expired-*.json`）。
- 右侧：该日期的只读任务表。
- 用于回顾过去某天的任务状态与已过期任务。

### 12.2 每日评估

**入口：** 工具栏 **每日评估**。

- 程序在每天 **23:59** 后自动生成当日任务完成情况评估（AI 或规则层）。
- 启动约 3 秒后会 **补评** 遗漏的日期。
- 对话框展示：完成数、到期数、未完结数、摘要与详细评价。
- 评估完成时，状态栏与托盘会通知。

## 13. 关闭与退出

### 13.1 点击标题栏 ×

弹出 **关闭** 对话框，三选一：

| 选项 | 说明 |
|------|------|
| **最小化到系统托盘**（默认） | 保存数据、隐藏窗口、托盘提示 |
| **退出** | 保存数据并完全退出 |
| **取消** | 关闭对话框，窗口保持打开 |

按 **Esc** 等同于 **取消**。

### 13.2 设置中的「关闭行为」

- **已勾选**「关闭窗口时最小化到托盘」：使用 **Alt+F4** 等系统关闭方式时，**直接** 最小化到托盘，不弹出三选一对话框。
- **未勾选**：关闭窗口将真正退出程序。

### 13.3 完全退出

- 托盘右键 → **退出**；
- 或在关闭对话框中选择 **退出**；
- 锁屏界面点击 **退出**。

## 14. 数据存储与日志

### 14.1 程序数据目录

路径：`<程序所在目录>/data/`

| 文件/目录 | 内容 |
|-----------|------|
| `tasks.db` | SQLite 主数据库 |
| `tasks-YYYY-MM-DD.json` | 每日任务快照 |
| `expired-YYYY-MM-DD.json` | 过期任务归档 |
| `top3-YYYY-MM-DD.json` | 当日 Top 3 及模型信息 |
| `logs/todolist-YYYY-MM-DD.log` | 应用运行日志 |
| `logs/ai-analysis-latest.log` | 最近一次 AI 分析完整记录 |

**迁移数据：** 复制整个程序文件夹（含 `data`）即可。

### 14.2 系统配置（注册表）

Windows 下配置保存在：

## 16. 常见问题

### Q1：AI 分析一直失败？

- 检查 **设置 LLM** 中 Provider 是否正确，点击 **测试连接**。
- Ollama 用户：确认服务运行且已 `ollama pull` 对应模型。
- 云端用户：检查 API Key 与网络。
- LLM 失败时仍可使用 **规则层 Top 3**，不影响基本待办功能。

### Q2：全局快捷键不生效？

- 打开 **设置 → 快捷键**，确认组合键并 **保存**。
- 检查是否与其他软件冲突（如输入法、截图工具）。
- 换一个组合键再试；状态栏会提示注册失败原因。

### Q3：忘记密码怎么办？

- 程序无「忘记密码」在线找回。可清除注册表中的 `security` 项或删除 QSettings 配置后重启（会丢失密码与相关锁屏设置，**不会** 删除 `data` 中的任务数据）。

### Q4：换电脑如何迁移？

- 复制整个绿色版文件夹（含 `data`）到新电脑即可。
- LLM / M365 等配置在注册表中，需重新设置，或一并导出注册表对应项。

### Q5：设置窗口显示不全？

- 设置对话框支持 **滚动**；也可调整窗口大小。
- 程序会根据屏幕分辨率自动限制最大高度。

### Q6：Microsoft 365 登录失败？

- 确认 Azure 应用已启用 **公共客户端/设备代码流**。
- 确认 Client ID 正确，且 API 权限包含 `Mail.Read`。
- 完成浏览器授权后，务必回到程序点击 **确认已授权**。
